# AGENTS.md

Operating manual for contributors and AI agents touching this repo. Read it before opening a PR.

This repo is a multi-course learning platform. The courses are the product. Every rule below keeps lessons coherent and the site functional across all courses.

---

## Repo layout

```
site/
  courses/
    <slug>/
      data.json                  ← course metadata (phases, lessons, glossary, prereqs)
      phases/
        <phase-slug>/
          <lesson-slug>/
            docs/en.md           ← lesson content
            quiz.json            ← 2 pre + 3 post questions
  combine.js                     ← build step: all data.json → data.js
  data.js                        ← generated; never edit by hand
  course.js                      ← runtime course resolver
  lesson.html / prereqs.html / catalog.html / glossary.html
  style.css
.claude/skills/generate-course/  ← Claude Code skill for generating new courses
```

---

## Hard rules

1. **Never edit `site/data.js` by hand.** It is generated by `node site/combine.js`. Always run that after touching any `data.json`.
2. **`contentHost` must be `"local"`** for any course whose lesson files live in this repo. Never set it to `"github"` if the files are here.
3. **Every `data.json` must have `prereqs` and `tierOrder`.** Without them the `/roadmap` page breaks for that course.
4. **Every lesson must have `docs/en.md` and `quiz.json`.** Stub files or placeholders are not acceptable — the site shows a 404 error.
5. **`path` field is required on every lesson.** Format: `phases/{course-slug}/{phase-slug}/{lesson-slug}`. Missing `path` breaks prev/next navigation.
6. **Fenced code blocks need a language tag.** Use `python`, `bash`, `json`, `text`, `mermaid`, etc.
7. **No marketing language.** No "powerful", "seamless", "revolutionary". Direct prose only.

## data.json schema rules

### Lesson paths

- `url` format: `https://github.com/{repo}/tree/{branch}/phases/{phase-slug}/{lesson-slug}/`
- `path` format: `phases/{course-slug}/{phase-slug}/{lesson-slug}`
- Phase slugs: `{id:02d}-{kebab-name}` — e.g. `00-foundations`
- Lesson slugs: `{index:02d}-{kebab-name}` — e.g. `01-variables-and-types`

### prereqs

```jsonc
"prereqs": {
  "0": [],       // keys are strings, values are arrays of integer phase ids
  "1": [0],
  "2": [1]
}
```

Every phase id must have an entry. Phase 0 is always `[]`.

### tierOrder

```jsonc
"tierOrder": [
  [0],        // tier 0: no dependencies
  [1],        // tier 1: depends on tier 0
  [2, 3]      // tier 2: parallel tracks shown side-by-side
]
```

Phases not listed are appended to the last row. The capstone phase always gets its own final tier.

## docs/en.md format

```markdown
# <Lesson Name>

> <One-line hook.>

**Type:** Build | Learn | Capstone
**Languages:** Python, Bash, etc.
**Prerequisites:** Phase N, Lesson NN — <name>
**Time:** ~XX minutes

## Learning Objectives
- <3–5 bullets, each starting with an action verb>

## The Problem
## The Concept
## Build It
## Exercises

## Key Terms
| Term | What people say | What it actually means |
```

- Minimum 400 words per lesson
- Every code block complete and runnable — no ellipsis, no placeholders
- ASCII diagrams for anything spatial

## quiz.json format

```jsonc
{
  "questions": [
    { "stage": "pre",  "question": "...", "options": ["A","B","C","D"], "correct": 0, "explanation": "..." },
    { "stage": "pre",  "question": "...", "options": ["A","B","C","D"], "correct": 2, "explanation": "..." },
    { "stage": "post", "question": "...", "options": ["A","B","C","D"], "correct": 1, "explanation": "..." },
    { "stage": "post", "question": "...", "options": ["A","B","C","D"], "correct": 3, "explanation": "..." },
    { "stage": "post", "question": "...", "options": ["A","B","C","D"], "correct": 0, "explanation": "..." }
  ]
}
```

Exactly 2 `pre` + 3 `post` questions. `correct` is zero-indexed. `explanation` must teach, not just confirm.

## Adding a new course

Use the Claude Code skill:

```
/generate-course Generate a "Python from Scratch" course. Slug: python, repo: user/python-from-scratch
```

Or manually:
1. Create `site/courses/<slug>/data.json`
2. Write `docs/en.md` + `quiz.json` for every lesson
3. Run `node site/combine.js`

---

## Validation

Before pushing, verify the build succeeds and the new course appears:

```bash
node site/combine.js
# Should print: ✅  Wrote site/data.js  (N courses, N phases, N lessons)

cd site && python3 -m http.server 4000
# Open http://localhost:4000 — new course should appear in the selector
```
